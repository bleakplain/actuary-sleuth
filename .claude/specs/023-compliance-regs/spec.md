# Feature Specification: 合规审查法规检索策略改造

**Feature Branch**: `023-compliance-regs`
**Created**: 2026-04-23
**Status**: Draft
**Input**: 改造合规审查法规检索策略，按险种加载专属法规库，强制包含通用法规和负面清单，确保法规覆盖完整性

## User Scenarios & Testing

### User Story 1 - 险种专属法规加载 (Priority: P1)

用户上传健康险产品文档，系统自动识别险种类型，加载健康险专属法规库（健康保险管理办法、短期健康险通知等），并结合通用法规库进行合规检查，确保法规覆盖完整。

**Why this priority**: 当前 RAG 检索可能遗漏关键法规，险种专属法规库是核心改进。

**Independent Test**: 上传已知健康险产品，验证检查结果中包含健康保险管理办法相关条款。

**Acceptance Scenarios**:

1. **Given** 用户上传健康险产品文档, **When** 系统识别险种为"健康险", **Then** 法规上下文包含健康保险管理办法、短期健康险通知等专属法规。
2. **Given** 用户上传寿险产品文档, **When** 系统识别险种为"寿险", **Then** 法规上下文包含人身保险精算规定、寿险产品指引等专属法规。
3. **Given** LLM 自动识别险种为"医疗险", **When** 用户确认险种类型, **Then** 使用确认后的险种加载法规库。

---

### User Story 2 - 通用法规强制包含 (Priority: P1)

所有产品的合规检查都强制包含保险法、条款管理办法、费率管理办法、精算规定等通用法规，不再依赖 RAG 检索是否命中。

**Why this priority**: 通用法规是所有产品的基础监管要求，必须确保覆盖。

**Independent Test**: 上传任意产品文档，验证检查结果中包含保险法、条款管理办法相关条款。

**Acceptance Scenarios**:

1. **Given** 用户上传任意保险产品文档, **When** 执行合规检查, **Then** 法规上下文必须包含保险法、条款管理办法、费率管理办法、精算规定。
2. **Given** RAG 检索未命中保险法条款, **When** 构建法规上下文, **Then** 仍强制包含保险法通用条款。

---

### User Story 3 - 负面清单独立检查 (Priority: P1)

负面清单作为独立检查项，在合规检查流程中强制包含，检查产品是否存在禁止性表述、禁止性条款等。

**Why this priority**: 负面清单是监管红线，必须独立检查且强制覆盖。

**Independent Test**: 上传包含禁止性表述的产品文档，验证负面清单检查项能识别问题。

**Acceptance Scenarios**:

1. **Given** 产品条款包含"保证续保"等禁止性表述, **When** 执行合规检查, **Then** 负面清单检查项识别并标记为 non_compliant。
2. **Given** 用户上传任意产品文档, **When** 执行合规检查, **Then** 检查结果中必须包含负面清单检查项（即使全部合规）。

---

### User Story 4 - 险种识别与确认 (Priority: P2)

系统通过 LLM 自动识别产品险种类型，用户确认后加载对应法规库；用户也可手动修正险种类型。

**Why this priority**: 提升用户体验，减少手动选择，同时保证准确性。

**Independent Test**: 上传产品文档，验证 LLM 能正确识别险种类型，用户可确认或修正。

**Acceptance Scenarios**:

1. **Given** 用户上传健康险产品文档, **When** LLM 分析文档内容, **Then** 自动识别险种为"健康险"并展示给用户确认。
2. **Given** LLM 识别险种为"医疗险", **When** 用户认为应为"重疾险", **Then** 用户可手动修正险种类型，系统使用修正后的险种加载法规库。
3. **Given** LLM 无法识别险种类型, **When** 系统提示用户手动选择, **Then** 用户手动选择险种后继续检查。

---

### Edge Cases

- 未识别到险种类型时如何处理？→ 提示用户手动选择，或使用通用法规库兜底
- 产品跨多险种（如健康险+意外险组合产品）如何处理？→ 支持多险种选择，合并法规库
- 负面清单检查项全部合规时如何展示？→ 显示"负面清单检查：全部合规"
- 法规库中部分法规未收录时如何处理？→ 标记缺失，提示用户补充

## Requirements

### Functional Requirements

- **FR-001**: 系统 MUST 支持按险种类型加载专属法规库（健康险、寿险、意外险等）
- **FR-002**: 系统 MUST 强制包含通用法规库（保险法、条款管理办法、费率管理办法、精算规定）
- **FR-003**: 系统 MUST 将负面清单作为独立检查项强制包含
- **FR-004**: 系统 MUST 支持 LLM 自动识别险种类型并让用户确认
- **FR-005**: 系统 MUST 支持用户手动修正险种类型
- **FR-006**: 系统 SHOULD 支持多险种组合产品的法规库合并
- **FR-007**: 系统 MUST 在法规上下文中标注法规来源类型（险种专属、通用、负面清单）

### Key Entities

- **险种类型**: 产品分类，决定专属法规库（健康险、寿险、意外险、医疗险、重疾险）
- **法规库分类**: 险种专属库、通用法规库、负面清单库
- **法规上下文**: 合规检查时使用的法规集合，包含来源标注
- **负面清单**: 禁止性规定列表，包含条款表述禁止、费率设定禁止、销售行为禁止等

## Success Criteria

- **SC-001**: 合规检查法规覆盖率 ≥ 95%（关键法规必须包含）
- **SC-002**: 险种识别准确率 ≥ 90%（LLM 自动识别）
- **SC-003**: 负面清单检查项覆盖率 100%（所有产品强制包含）
- **SC-004**: 用户确认险种后检查延迟 < 2s（法规库加载时间）

## Assumptions

- 险种专属法规库已在知识库中标记分类（需确认现有知识库是否有分类信息）
- 负面清单表结构已支持条款禁止、费率禁止等分类（已存在 negative_list 表）
- LLM 可从产品文档内容中提取险种类型信息
- 用户了解保险产品险种分类概念
